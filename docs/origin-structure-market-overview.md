# Origin Structure Market — Overview

## 1. Concept

Origin Structure Market is a protocol-level framework for registering, tracing, and monetizing AI-era intellectual structures.

It does not treat ideas as static content.
Instead, it treats them as **structured origin assets** that can evolve, branch, and generate derivative value across AI systems, creators, and applications.

The core principle is:

> Value should flow back to the structural origin, not only the final product.

---

## 2. Core Problem

Modern digital platforms suffer from a structural imbalance:

* Ideas are reused without attribution
* Derivative works obscure original contributors
* AI systems amplify content without traceable origin
* Value is captured at the platform layer, not the source layer

This creates a broken value loop where:

> Creation ≠ Compensation

---

## 3. Proposed Solution

Origin Structure Market introduces a protocol-based solution:

### 3.1 Origin Asset Registration

Ideas are registered as **Origin Assets**:

* Structured definition (not raw text only)
* Canonical hash (immutability)
* Timestamped origin proof
* Permission + royalty policy
* Traceable references

---

### 3.2 Derivative Tracking

All downstream outputs can declare:

* Which origin assets they are derived from
* Contribution weights
* Transformation type (prompt, schema, GPT, article, etc.)

This creates a **traceable lineage graph** of ideas.

---

### 3.3 AI Multi-Wing Expansion

AI agents operate as "Wings":

* Fire Wing: generation and expansion
* Wind Wing: distribution and lightweight propagation
* Water Wing: consistency and reasoning validation
* Earth Wing: governance, contracts, and royalty logic

Ideas are not static—they are **biologically expanded structures**.

---

### 3.4 Royalty Allocation Layer

When a final product generates revenue:

* Revenue is distributed across:

  * Final creator
  * Derivative creators
  * Origin creators
  * Auditors / evaluators
  * Protocol maintenance fund

This enables **reverse value flow to the origin layer**.

---

## 4. System Architecture

Origin Structure Market is composed of:

* Origin Asset Schema
* Derivative Asset Schema
* Origin Audit Layer
* Royalty Allocation Graph
* Trace Receipt Standard integration
* AI evaluation subsystem (future)

It functions as a **meta-layer above creative ecosystems**, not a platform itself.

---

## 5. Minimal Viable Product (v0.1)

The first implementable version includes:

* JSON Schema validation for Origin Assets
* YAML-based registration examples
* GitHub Actions validation pipeline
* Static trace definitions

No blockchain is required at this stage.

The system begins as:

> A verifiable structural registry.

---

## 6. Design Philosophy

This system is based on three principles:

### 6.1 Structure over content

What matters is not the text itself, but its reproducible structure.

### 6.2 Trace over ownership

Ownership is derived from traceable contribution, not assertion.

### 6.3 Flow over storage

Value is defined by how it moves through systems, not where it is stored.
